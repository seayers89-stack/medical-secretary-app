Accessibility statement

Last updated: 3 July 2026

This statement applies to the Veyn website at www.veyn.uk, operated by VeynUK Ltd.

We want Veyn to be usable by as many people as possible. We are working to make this website accessible in accordance with the Web Content Accessibility Guidelines (WCAG) 2.1.

1. Compliance status

Partially conformant

This website is partially conformant with WCAG 2.1 level AA. "Partially conformant" means that some parts of the content do not fully conform to the accessibility standard. We are actively working to improve conformance across the site.

2. What we have done

  • Semantic HTML elements used throughout (headings, lists, buttons, labels)
  • Visible focus states on interactive elements
  • Sufficient colour contrast between body text and background (verified for primary text)
  • All form fields have associated labels
  • Pages use a logical heading hierarchy (h1 → h2 → h3)
  • Language attribute set on all pages (lang="en")
  • No content that flashes or blinks more than three times per second
  • Navigation is consistent across the site

3. Known issues

We are aware of the following accessibility limitations and are working to address them:

  • SVG icons: Some decorative SVG icons do not have explicit aria-hidden="true" attributes, which may cause screen readers to announce them unnecessarily.
  • Keyboard navigation in modals: Focus trapping within modal dialogs (e.g. the cookie consent banner) may not behave ideally with all screen readers.
  • Colour contrast — gold accents: The gold accent colour (#B8924A) used for links and labels on our cream background (#F7F4ED) has not been independently verified to meet WCAG AA 4.5:1 ratio for all text sizes; we are reviewing this.
  • Skip navigation link: A "skip to main content" link is not yet present on all pages.
  • Dynamic content announcements: Some status messages (e.g. form success/error states) are not yet announced via ARIA live regions.

4. What we are doing to improve

We are committed to improving accessibility over time. Planned improvements include:

  • Adding aria-hidden="true" to all decorative icons
  • Adding ARIA live regions for dynamic status messages
  • Reviewing and confirming colour contrast ratios across all UI components
  • Adding a "skip to main content" link on all pages
  • Conducting keyboard-only navigation testing across all account pages

5. Reporting accessibility problems

If you find any accessibility problems not listed here, or if you need information in an alternative format, please contact us. We aim to respond to accessibility feedback within 5 working days.

Contact: infoveynuk@gmail.com

Please describe the page you were visiting, the problem you encountered, and (if applicable) the assistive technology you were using.

6. Enforcement

Veyn is operated by a private company and is not legally required to comply with the Public Sector Bodies Accessibility Regulations 2018. However, we are committed to meeting WCAG 2.1 AA as best practice under the Equality Act 2010, which requires reasonable adjustments to avoid placing disabled people at a substantial disadvantage when accessing services.

If you are not satisfied with our response to your accessibility concern, you may contact the Equality Advisory and Support Service (EASS).

7. Technical information

Veyn is built with HTML, CSS, and JavaScript. The site is hosted on GitHub Pages (static files) with a Supabase backend. There are no third-party widgets loaded except Stripe (payment iframes) and Google Fonts.
